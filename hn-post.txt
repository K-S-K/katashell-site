Hi HN. I've been writing software for about thirty years, most of
it on Windows (C#/.NET, before that C++, some embedded). Two years
ago I moved to a Mac and liked almost everything about it — except
that no file manager worked the way my hands expected: a directory
tree on the left, a details view with permissions and dates,
function keys that do things, and a terminal one keystroke away.

So I built one. Kata Shell is a native macOS app: Swift 6, AppKit
plus SwiftUI, no Electron and no web views anywhere. The directory
tree is an NSOutlineView, the file list is an NSTableView, the
terminal is SwiftTerm running a real local shell session that
follows the folder you're browsing (it tracks your cd via OSC 7,
and ignores it when the shell is remote, so an SSH session can't
spoof the path). Markdown preview renders as native SwiftUI views
via MarkdownUI.

A couple of things that might be interesting technically:

The first version was actually .NET MAUI Blazor Hybrid, because
that's my home stack. It reached feature parity and then I threw
it away: the WebKit/Mac Catalyst bridge produced a steady stream
of quirks that cost more to work around than a rewrite. The Swift
version is faster, smaller, and has had far fewer surprises.

It's not in the Mac App Store, deliberately. MAS requires the App
Sandbox, and a sandboxed file manager can't browse your disk
without per-folder grants — and a sandboxed embedded shell
inherits the sandbox, which makes it mostly decorative. So it's
Developer ID signed and notarized instead, distributed as a DMG.

No analytics, no telemetry; the only network call is the Sparkle
update check, which you can turn off.

It's free during early access; 1.0 will be a one-time 29 EUR
license (a year of updates, the version you have is yours
forever — no subscription). Early-access users get a 1.0 license
free.

I'm the target audience for this tool, which means I'm also blind
to what other people need first. If you came to the Mac from
Windows — or if you didn't and this still looks useful — I'd
honestly like to hear what's missing or broken.
